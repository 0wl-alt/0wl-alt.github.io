梆梆加固特征

​ 在 apk 的 lib 目录下可以发现这两个 so 文件

原理分析

  • 梆梆加固免费版的旧版本是二代壳，采用的是内存不落地加载，在运行过程中将stubApplication 替换为真正要执行的 Application ，然后转到真正的 onCreate
  • 可以看到 java 代码中调用 System.loadLibrary() 加载 native 库，在attachBaseContext 这个函数内加载了 realApplication （时间早于 oncreate）
